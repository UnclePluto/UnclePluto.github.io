公司的旅游终于定下来了~

闲谈

这周五晚上，收到助理的邮件，拖了 1 年多的旅游计划终于有着落了，也可以趁着这次公司的旅游出国去看看了，有去日本的团，不过不知道排不排的上号。

因为这个契机也把一直打算学而一直没好好学的日语下定决心去学习下，不再万年五十音了。我觉得日语还是要学习下，无论对这次旅游而言，还是对今后的发展来说，都是有些必要的。

对于旅游而言，会基础的交流语言以便应急，我觉得完全无法沟通也是不安全的，虽说日本人也会说英文，但是真心害怕听不懂他们的英文。然而对于今后的发展来说，首先做技术的英文好的优势就不用说了。基本上所有的技术文档都是有英文版，最新的技术也都是会以英文陈述。而日语我觉得是后续中国一些技术方案走国际化的一个方向，印度那边当然也是了，不过我觉得日本是一个更有经济基础，且互联网与 O2O 这方面严重滞后的国家，说以中国很多成功的经验以后可以尝试去日本复制，而且日本也有这个经济基础去接受这些改变。

学习

本周开始学习了 python，总体了解了语法的一些知识，有些比较新奇的数据结构，比如元组，其实我一直都很好奇这类的数据结构，比如 OC 的不可变数组等等，就是专门给不可变单做一套数据结构，但是我们实际上常常用到的却都是变量，那么这些不可变的数据类型都用常量。虽然我知道在实际的项目中，会有很多这类的需求，而且可能从底层的资源分配来说，也决定了这类不可变数据类型的存在。我只是觉得吧，如果我去设计一套语言，可能就会真的把这类不可变的静态类型尽可能的少。

就目前的学习进度来看，我觉得 python 跟 php 的差距并不是很大，python 最大的优势可能还是在于优质的三方库众多，感觉 php 的三方库也就是有了 compser 以后才稍微好用点，不过还是有大批量的 php 开发者完全不知道这个东西，还在用面向过程的方式去写网站。其实这也就是为啥 php 老被黑，他太过于容易，可以让完全没有一点编程思想的人就能在短时间做出一个 web 网站，这个网站可能有各种各样的问题与不足，但是这并非语言的问题，而是人的问题。当然了这并不是说 php 语言本身没有任何缺点，只是说大部分黑 php 的人，黑的点都不是 php 语言本身。

些废话也能写这么多，不过话说回来，新买的青轴机械键盘，就应该多打打字！
